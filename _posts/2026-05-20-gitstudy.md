# Git 병합(Merge)과 충돌(Conflict)

## 병합(Merge)이란?

병합(Merge)은 서로 다른 브랜치의 작업 내용을 하나로 합치는 작업이다.

브랜치를 생성하는 이유는 원본 코드(main/master)에 직접 영향을 주지 않고 독립적으로 개발하기 위해서이다.

즉:

- 기능 개발
- 오류 수정
- 실험 코드 작성

등을 안전하게 진행하기 위해 브랜치를 분리한다.

이후 개발이 완료되면 원본 브랜치에 작업 결과를 반영해야 하는데, 이 과정을 병합(Merge)이라고 한다.

---

# 병합 방식

## 1. 수동 병합

코드 두 개를 직접 비교하면서 수정하는 방식이다.

---

## 2. 자동 병합

Git이 변경 사항을 분석하여 자동으로 병합하는 방식이다.

예시:

```bash
git merge feature
```

---

# 정리

Git 병합은 여러 브랜치의 작업 내용을 하나로 합치는 매우 중요한 기능이다.

하지만 같은 파일의 같은 위치를 수정하면 충돌이 발생할 수 있다.

따라서:

- 브랜치 관리
- 주기적인 동기화
- 팀 규칙

등이 중요하며, 충돌이 발생했을 때는 직접 코드를 수정하여 해결해야 한다.
